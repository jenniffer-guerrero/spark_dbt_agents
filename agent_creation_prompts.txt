PROMPT 1 — Create Shared Skills File (.agent-skills.md)

Create or replace the file `.agent-skills.md` in the repository root.
Use YAML frontmatter with a `skills` section.
The file content must be entirely in English.

Required skill groups and scope:

1) dbt
- Create dbt models compatible with Databricks SQL.
- Follow layer conventions: silver (1), gold/feature_set (20), gold/export (29).
- Use reusable macros where appropriate.
- Document lineage and dependencies.

2) spark
- Implement equivalent transformations with PySpark or Spark SQL.
- Keep logic modular, testable, and runtime-compatible.

3) incremental_mapping
- Design delta-driven recomputation for REL_ID mapping using graph expansion from changed nodes only.
- Define and maintain state snapshots: edge_snapshot, seed_snapshot, seed_membership.
- Apply deterministic winner selection when multiple seeds reach one rel_id:
  - minimum numeric seed when available
  - lexical minimum as fallback

4) notebook_engineering
- Separate update logic notebook from scenario-based test notebook.
- Document notebook flow with phase comments and architecture diagrams.
- Support JSON-based notebook code loading when nbformat is unavailable.

5) spark_runtime_reliability
- Use staging-table swap writes when UPDATE/DELETE support is limited on local Parquet tables.
- Apply checkpointing and cache invalidation to reduce lineage/stale-plan issues.
- Include handling guidance for local metastore and Java runtime constraints.

6) stateful_test_design
- Build sequential mutation scenarios over shared shebang tables.
- Validate exact inserted/removed deltas in output.
- Verify output dedup invariants on (REL_ID, REL_ID_REGIE_KLANT).
- Include baseline snapshot/restore strategy for safe replay.

Required body sections:

A) Agent Identity
- You are the test_data agent for Spark SQL incremental mapping validation.

B) Goals

  - ggm_np: new rel_id/ikb_no links, rel_id reassignment, ikb_no reassignment
- Preserve deterministic winner rule for conflicting seeds:
  - minimum numeric seed wins

C) Core Skills
- Sequential scenario engineering.
- Delta-first assertions.
- Impact-focused validation.
- Notebook observability guidance (full-table display for debugging; strict assertions for pass/fail).

D) Outcome Evaluation Protocol
- Compare expected vs actual deltas.
- Report pass/fail per scenario.
- Explain failures and suggest fixes.
- Classify likely cause:
  - logic defect
  - data precondition issue
  - environment/runtime issue

E) Constraints
- Keep scenarios minimal and reproducible.
- Assume stateful runs where shebang.output evolves after each scenario.
- Use English in outputs.

F) Recommended Scenario Catalog
- Seed creation/removal in direct_bank.
- Relationship rewiring in ggm_np.
- Duplicate source-row robustness.
- Simultaneous cross-table insertions sharing one identifier.
- No-op reruns for idempotency proof.
